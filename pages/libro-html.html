Meta Viewport

El primer tipo de metadatos que veremos es el primero que vemos al crear el documento HTML,

Bien empecemos explicando el metadato paso por paso, primero tenemos la etiqueta en si "meta" la cual ya sabemos su funcionamiento, despues tenemos la propiedad "name" con el valor "viewport", la propiedad "name" define el tipo de metadato, en este caso el metadato es "viewport", este metadato le dice al navegador como debe mostrar y escalar la pagina en dispositivos moviles.

Por defecto, antes de HTML5, los navegadores moviles solian simular un ancho de pantalla fijo (ejemplo: 980px) y luego reducian el contenido para que encajara en pantallas peque침as. esto hacia que los titios se vieran muy peque침os o ilegibles en celulares.

El metadato "viewport" cambia ese comportamiento, permitiendo controlar directamente como se adapta la pagina al ancho real del dispositivo.

"width=devicewidth" Hace que el ancho del 치rea visible (viewport) sea igual al ancho f칤sico del dispositivo. Esto permite que el dise침o responda al tama침o real de la pantalla. En un iPhone de 390px, el viewport tambi칠n ser치 de 390px.

"initial-scale=1.0" Define el nivel de zoom inicial cuando se carga la p치gina. Un valor de 1.0 significa que no hay zoom (la p치gina se muestra al 100%). Si pones initial-scale=2.0, la p치gina se ver치 ampliada al doble.

Puedes agregar mas valores dentro de "content", separados por comas, por ejemplo:

"minimum-scale=1.0" Esto evita que el usuario haga zoom para alejar mas de ese valor.

"maximum-scale=1.0" Esto evita que el usuario haga zoom para hacercar mas de ese valor.

"user-scalable=no" Esto desactiva completamente el zoom tactil, esta opcion no es recomendada.
